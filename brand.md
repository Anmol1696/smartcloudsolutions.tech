# Brand - SmartCloud Product Lab

_Status: active - v3 (2026-05-13)_

The visible wordmark for `smartcloudsolutions.tech` is `SmartCloud`. The legal
entity, `Smart Cloud Solutions FZ-LLC`, appears in the footer only. The site
speaks as a small senior engineering lab: concrete, direct, proof-led.

## Design Direction - Product Lab

A restrained, light developer-tool site for a 2-engineer studio. It should feel
closer to a product surface than an agency mood board: clean sans typography,
high-contrast copy, compact proof, and interface-like artifacts built from real
project evidence.

| Field | Value |
| --- | --- |
| Theme | Light only. Near-white canvas, dark graphite text, cool technical accent. |
| Density | Compact and composed. No giant theatrical gaps. |
| Surface | Product panels, rows, and artifact strips. Avoid repeated marketing cards. |
| Type | Geist Sans for UI and display. Geist Mono only for numbers and tiny metadata. |
| Motion | 100-180ms property-specific transitions. No page theatrics. |
| Do | Outcome-led work rows, product artifacts, tight section rhythm, strong sans. |
| Don't | Yellow/amber, serif display, `//` labels, terminal trees, decorative code, card soup. |

## Typography

- **All display and UI:** Geist Sans. Use weight and scale, not serif contrast.
- **Mono:** Geist Mono only for stat values, package names, timestamps, and short
  technical metadata.
- **Headlines:** direct and compact. No italic phrases, no negative tracking.

## Voice

Senior-engineer-to-senior-engineer. Name the artifact, the operating failure,
and the outcome. Avoid agency filler like "unlock potential" and template
phrases like "build the future."
